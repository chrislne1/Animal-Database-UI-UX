# Animal-Database-UI-UX  
**UI/UX Application | Database Management | Front-End & Back-End Integration**

## Overview  
Animal-Database-UI-UX is a **full-stack web application** designed to manage and visualise a dynamic animal dataset, developed to showcase **UI/UX engineering**, **database design**, and **interactive data presentation**.  

The project integrates **responsive front-end interfaces**, **RESTful API endpoints**, **relational database schema**, and **data-driven UI components** into a modular, scalable application framework.

---

## Tech Stack  
- **Languages:** JavaScript (ES6+), HTML5, CSS3  
- **Frameworks/Libraries:** React (UI components), Redux (state management), Node.js/Express (API server)  
- **Tools:** Git, NPM/Yarn, Webpack/Babel, Postman (API testing)  
- **Database:** Relational (e.g., PostgreSQL or MySQL) with schema for animal records, metadata, user interactions  
- **Assets:** High-resolution animal imagery, metadata JSON/CSV files, UI icon sets  
- **Persistence:** CRUD operations via REST API, database transactions, user session state  

---

## Core Features  
- **Responsive UI/UX Design:** Adaptive layouts, intuitive navigation, data-table views, card grids.  
- **Front-End State Management:** Global state via Redux, asynchronous actions for fetching/updating data.  
- **Back-End API Engine:** Express routes for animal data retrieval, insertion, update and deletion.  
- **Relational Data Model:** Structured tables for Animals, Species, Habitats, Users, with normalized schema.  
- **Interactive Data Visualization:** Filtering, sorting, searching, pagination of animal records.  
- **User Session Management:** Authentication/authorization scaffolding, user roles, secured endpoints.  
- **Asset Integration:** Embedded image galleries, metadata links, downloadable data exports.  

---

## Design & Techniques  
- **Component-Based Architecture:** Modular React components (e.g., AnimalCard, FilterPanel, DetailModal).  
- **Flux/Redux Pattern:** Single-source state, dispatching actions, reducers to update views.  
- **RESTful API Design:** Clean HTTP verbs, structured JSON payloads, status codes and error handling.  
- **Database Normalisation:** Use of primary/foreign keys, relationships (one-to-many, many-to-many), indexing.  
- **File I/O & Data Import:** Support for importing CSV/JSON datasets into the database, data validation routines.  
- **UI/UX Best Practices:** Accessibility (ARIA roles), responsive breakpoints, consistent styling and theming.
